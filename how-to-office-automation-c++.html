Office Automation in C++

Sat 20 September 2014
By Sagar Gandhi

This post is about Microsoft Office Automation in C++. There are almost no articles about how to acheive office automation in raw COM, C++; there are lot of suggestion about doing same thing in C# or in general .Net languages. This post is for people who have come to conclusion that they want to do office automation in C++. Its not worth a point which one is good, it all depends on your project requirement.

This post also assumes that you are
1. quite familier with C++ AND COM style of coding,
2. in general what automation is
3. Its challenging and fun to do it, be ready.

Here we will consider example of Microsoft Powerpoint. Idea is almost same for all the office products, things are little different for Outlook, but this post is still applicable for that also.

First, you are going to need Type Library for the product whose automation you want to acheive. These type libraries have extension .tlb or .olb. You can use OLE COM Object Viewer to view contents of this file. Actually, this is enough for writing automation code. But I am going to present quite easier method, with instructions about how-to in detail, but with very less code. So your first task is finding this .tlb or .olb file as per your needs. For Powerpoint this file is called MSPPT.OLB.

Next, we are going to use #import with few flags.
